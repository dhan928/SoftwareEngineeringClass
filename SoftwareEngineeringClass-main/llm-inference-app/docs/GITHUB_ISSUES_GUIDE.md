# Change Log & GitHub Issues Guide

## 1. change.log expectation

The repo root now includes a `change.log` file. Fill in the real PR numbers from your GitHub repository before submission.

Format used:
- feature / task
- files changed
- pull request reference
- short note

## 2. GitHub issues expectation

I cannot create or backfill GitHub issues from this local environment, so you still need to make sure your real repository has:
- issues for the iteration 2 features
- discussion/comments showing dev activity
- issue references in your PRs
- issue closure when the work is done

## Suggested issues to have in GitHub

1. Implement conversation search
2. Implement resume existing conversation
3. Add Jasmine unit tests for conversation flows
4. Update docs for architecture, API, and database design
5. Add/change change.log entries for iteration 2

## Suggested PR naming

- `feat/search-conversation-history`
- `feat/resume-conversation-thread`
- `test/add-jasmine-conversation-service-tests`
- `docs/add-iteration2-dev-report-artifacts`

## What to submit in the report

You can cite:
- the files in `docs/`
- the `change.log`
- the Jasmine test files in `backend/tests/`

But for GitHub issues specifically, you still need your real repo links or screenshots.
